# PM 头脑风暴 Skill

把产品经理的核心思维封装进 AI，通过对话式引导帮你把「有个想法」变成「想清楚了的产品方向」。

---

## 这个 Skill 是干什么的

很多人在做产品时会踩同一个坑：**还没想清楚方向，就开始写需求文档或者直接开发**。方向没对齐，后面所有的细节都是浪费。

这个 skill 的定位是**想清楚用的**，不是写文档用的：

- 帮你想清楚产品是给谁用的、解决什么问题
- 帮你判断这件事值不值得做
- 帮你在多个方向之间做出有依据的选择
- 帮你确定 MVP 范围，知道先做什么、不做什么

想清楚之后，再用 **prd-writer** 把方向展开成完整的产品需求文档。

```
pm-brainstorming（想清楚）→ prd-writer（写出来）→ 开发实现
```

## 适合谁用

| 用户类型 | 用法 |
|---------|------|
| **产品经理** | 用结构化流程替代漫无目的的思考，快速对齐方向 |
| **开发者/工程师** | 有想做的东西但不知道从哪里想起，用 PM 视角补盲区 |
| **创业者** | 验证一个新方向是否值得投入资源 |
| **运营/业务同学** | 有业务需求但不知道如何转化成产品思路 |

- 「分析一下竞品」
- 「这几个方案怎么选」
- 「帮我想想这件事能不能做」

## 核心流程

按 PM 真实思考顺序排列，9 步走完，输出一份产品概念文档。

```
步骤 1  理解背景（含范围评估）
步骤 2  判断成熟度
步骤 3  目标用户深挖          ← 最关键的一步
步骤 4  竞争定位分析
步骤 5  商业视角验证          ← 内部/自用工具可跳过
步骤 6  开发视角 + 产品形态
步骤 7  提出 2-3 个产品方向
步骤 7b 关键假设识别          ← 可选，有不确定前提时触发
步骤 8  功能方向与优先级
步骤 9  产出概念文档 → 自查 → 用户确认 → 过渡
```

---

## 产出物：产品概念文档

脑暴结束后输出一份结构化的产品概念文档，包含：

- **一句话定位** — 给谁用、解决什么问题、核心差异是什么
- **产品目标** — 对用户和对做产品的人各自的价值
- **产品形态** — Web / 小程序 / App / AI Skill，以及选择理由
- **目标用户与使用场景** — 用户画像、典型场景、现有替代方案
- **竞争分析摘要** — 竞品格局和差异化定位
- **商业模式** — 预计怎么赚钱（内部/自用项目省略）
- **核心功能方向** — 按优先级分类（必须有 / 重要 / 差异化亮点）
- **不做什么** — MVP 边界
- **关键假设** — 方向成立的前提条件（可选）
- **关键决策记录** — 脑暴过程中做的选择和理由
- **待确认问题** — 还需要验证的事项
- **下一步建议** — 是否继续调用 prd-writer

## 设计理念

**一次只问一个问题**
不会一口气抛出十个问题让你应接不暇。每次只问最重要的那一个。

**先验证方向，再展开细节**
方向没对齐前，不展开任何细节。方向错了，细节全部推倒重来。

**PM 思维框架，不需要你懂框架**
用户任务视角、竞品定位、功能优先级分类等方法论已经嵌入引导问题里，你只需要回答，不需要知道那些框架叫什么名字。

**分节呈现，逐步确认**
每个阶段的内容分节输出，每节确认 OK 后再继续，不会一口气全输出然后问你「大概怎么样」。
